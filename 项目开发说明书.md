## 二、数据库设计

### 2.1 数据表结构

#### TASKS - 任务表

```sql
SELECT PROCESSNAME, INCIDENT, STEPID, STEPLABEL, ASSIGNEDTOUSER, STATUS
FROM TASKS
```

说明：
- `PROCESSNAME` - 流程名称
- `INCIDENT` - 实例号
- `STEPID` - 步骤ID
- `STEPLABEL` - 步骤名称
- `ASSIGNEDTOUSER` - 分配给用户 (格式: DOMAIN/UserID)
- `STATUS` - 状态 (1=进行中)

用户ID提取方式：从 ASSIGNEDTOUSER 字段中提取 `/` 后面的部分

#### UnfinishedTask - 未完成任务表

```sql
SELECT PROCESSNAME, INCIDENT, stepid, STEPLABEL, ASSIGNEDTOUSER, status, startdate, enddate, newenddate
FROM UnfinishedTask
```

字段说明：
| 字段 | 说明 |
|------|------|
| PROCESSNAME | 流程名称 |
| INCIDENT | 实例号 |
| stepid | 步骤ID |
| STEPLABEL | 步骤名称 |
| ASSIGNEDTOUSER | 分配给用户 |
| USERID_AFTER_SLASH | 提取后的用户ID |
| status | 状态 (pending/in_progress/completed) |
| startdate | 开始日期 |
| enddate | 结束日期 |
| newenddate | 延期日期 |

#### worklog - 工作日志表

```sql
CREATE TABLE worklog (
    id INT AUTO_INCREMENT PRIMARY KEY,
    INCIDENT INT,
    stepid INT,
    workdate DATE,
    spendtime INT
)
```

### 2.2 日期字段更新规则

| 字段 | 更新条件 |
|------|---------|
| startdate | 只能设置一次，无值时允许设置 |
| enddate | startdate 有值且 enddate 为空时允许设置，必须大于 startdate |
| newenddate | startdate 和 enddate 都有值时允许设置，必须大于 enddate，可多次修改 |

### 2.3 日期验证逻辑

```
startdate: 不能大于 enddate，不能大于 newenddate
enddate: 必须大于 startdate，不能大于 newenddate
newenddate: 必须大于 startdate，必须大于 enddate
```

## 三、功能设计

### 3.1 用户登录

- **路由**: `/login` (GET/POST)
- **功能**: 根据 userid 登录，验证用户是否存在
- **验证逻辑**: 查询 TASKS 表中 SUBSTRING(ASSIGNEDTOUSER, CHARINDEX('/', ASSIGNEDTOUSER) + 1, LEN(ASSIGNEDTOUSER)) = userid
- **排除用户**: ASSIGNEDTOUSER <> 'SYSTEMUSER'
- **临时方案**: 使用 user100 格式登录

### 3.2 工时记录

- **路由**: `/work` (GET/POST)
- **功能**:
  - 显示当前用户的进行中任务 (STATUS = 1)
  - 同一任务可记录多条工时
  - 支持添加工时记录
  - 支持修改工时记录
  - 支持删除工时记录
  - 显示工时合计

### 3.3 任务管理

- **路由**: `/task` (GET/POST)
- **功能**:
  - 显示当前用户所有任务
  - 更新任务状态 (pending/in_progress/completed)
  - 按规则更新日期字段 (startdate/enddate/newenddate)

## 五、核心代码逻辑

### 5.1 用户ID提取

```python
def extract_userid(assigned_to_user):
    if '/' in assigned_to_user:
        return assigned_to_user.split('/')[-1].strip()
    return assigned_to_user.strip()
```

### 5.2 登录验证SQL

```sql
SELECT DISTINCT ASSIGNEDTOUSER
FROM TASKS 
WHERE SUBSTRING(ASSIGNEDTOUSER, CHARINDEX('/', ASSIGNEDTOUSER) + 1, LEN(ASSIGNEDTOUSER)) = %s
  AND ASSIGNEDTOUSER <> 'SYSTEMUSER'
```

### 5.3 工时记录查询SQL

```sql
SELECT PROCESSNAME, INCIDENT, STEPID, STEPLABEL, ASSIGNEDTOUSER, STATUS
FROM TASKS 
WHERE SUBSTRING(ASSIGNEDTOUSER, CHARINDEX('/', ASSIGNEDTOUSER) + 1, LEN(ASSIGNEDTOUSER)) = %s
  AND STATUS = 1
  AND ASSIGNEDTOUSER <> 'SYSTEMUSER'
ORDER BY INCIDENT, STEPID
```

### 5.4 任务管理查询SQL

```sql
SELECT PROCESSNAME, INCIDENT, stepid, STEPLABEL, ASSIGNEDTOUSER, status, startdate, enddate, newenddate
FROM UnfinishedTask 
WHERE RTRIM(USERID_AFTER_SLASH) = %s
ORDER BY INCIDENT, stepid
```
